## 4. 各阶段详细计划

## Phase 0：验证平台起步与骨架烟测

### RTL完善
- 冻结当前 `Baseline-Alpha`
- 修正明显阻塞缺陷（编译/联调阻塞项）

### UVM验证
- 搭建最小 UVM 框架：`tb_top/env/base_test/smoke_test`
- 建立 APB agent（主）与最小寄存器访问序列
- 跑通 `tc_smoke_boot_apb`

### 交付物
- 基础 filelist 与 run 脚本
- 第一版 smoke 回归报告

### Exit Criteria
- smoke 连续 3 次通过
- 无阻塞级（P0）问题

### 完成状态（2026-04-08）
- `smoke_apb_test` 已完成稳定回归，满足“连续 3 次通过”目标
- 当前无阻塞级（P0）问题阻碍进入 Phase 1
- Phase 0 交付物已具备：最小 UVM 链路、基础 filelist、统一 run 脚本

---

## Phase 1：关键功能闭环（I2C + UDMA + APB）

### RTL完善
- I2C：补齐基础控制流程（start/stop/repeated-start 框架）
- UDMA：完善通道控制、完成中断、边界长度行为
- APB寄存器：复位值与字段行为统一

### UVM验证
- I2C 基本读写、repeated-start
- UDMA 基础搬运与中断检查
- APB R/W、reset value 检查
- 引入基础 scoreboard 与 predictor

### 交付物
- 用例：`tc_i2c_basic_rw`、`tc_i2c_repeat_start`、`tc_udma_basic`
- 缺陷清单与修复闭环记录

### Exit Criteria
- 关键功能用例全部通过
- 功能覆盖率达到阶段目标（>70%）

---

## Phase 2：系统闭环（中断 + 低功耗 + E2E）

### RTL完善
- PMU/RTC/WAKE 路径一致性
- INTC 的屏蔽/挂起/清除路径完善
- 顶层中断映射与唤醒原因寄存器联动

### UVM验证
- `RUN/SLEEP/DEEP-SLEEP` 切换与唤醒
- 多中断并发与优先级行为
- 端到端场景：I2C采集 -> UDMA搬运 -> 融合事件 -> 中断/上报

### 交付物
- 低功耗场景回归报告
- E2E 用例报告

### Exit Criteria
- 低功耗关键场景通过
- E2E 稳定通过（多seed）

---

## Phase 3：异常/随机/覆盖收敛

### RTL完善
- 补齐异常路径（NACK/timeout/busy冲突）
- 修复随机回归暴露的 corner case

### UVM验证
- 异常注入测试
- 大规模随机回归
- 功能覆盖 + 代码覆盖收敛

### 交付物
- 覆盖率报告（func/code）
- 未覆盖项说明与风险评估

### Exit Criteria
- 阶段覆盖目标满足（示例：func>90%, line>90%, fsm>95%）
- 无未关闭P0/P1

---

## Phase 4：实现签收与封板

### RTL完善
- 清理综合告警与不可综合结构
- 对齐综合约束与形式检查假设

### 验证/实现
- DC 综合 QoR
- FM 等价检查
- ECO 后最小回归

### 交付物
- 综合报告、FM报告、签收清单

### Exit Criteria
- FM 等价通过
- 阶段签收评审通过
