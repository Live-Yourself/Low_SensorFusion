# 骨架版到完整版：分阶段开发与验证计划（UVM-SFSoC-v1）

## 1. 文档目的

基于当前“可联调骨架版”RTL，制定从 `Alpha` 到 `Full` 的阶段化路线，明确：
- 每阶段 RTL 完善项
- 每阶段 UVM 验证项
- 进入下一阶段的准入标准（Exit Criteria）

---

## 2. 当前基线定义（Baseline-Alpha）

当前状态（已具备）：
- SoC 顶层 APB 互连与读回 mux
- I2C/UART/SPI/RTC/PMU/GPIO/TIMER_WDT/UDMA/INTC/FUSION 的 IP 骨架
- 统一目录分层（`pkg/reg_top/core/top`）

当前状态（未完成）：
- 协议细节与异常路径完备
- 低功耗时序细化与可证明正确性
- UVM 环境、RAL、覆盖率闭环
- 综合/形式验证签收闭环

---

## 3. 分阶段总览

- Phase 0：验证平台起步与骨架烟测（1~2周）
- Phase 1：关键功能闭环（I2C + UDMA + APB）（2~3周）
- Phase 2：系统功能闭环（中断 + 低功耗 + E2E）（2~3周）
- Phase 3：异常/随机/覆盖收敛（2~4周）
- Phase 4：实现签收（DC/FM）与项目封板（1~2周）

---

## 4. 各阶段详细计划

## Phase 0：验证平台起步与骨架烟测

### RTL完善
- 冻结当前 `Baseline-Alpha`
- 修正明显阻塞缺陷（编译/联调阻塞项）

### UVM验证
- 搭建最小 UVM 框架：`tb_top/env/base_test/smoke_test`
- 建立 APB agent（主）与最小寄存器访问序列
- 跑通 `tc_smoke_boot_apb`

### 交付物
- 基础 filelist 与 run 脚本
- 第一版 smoke 回归报告

### Exit Criteria
- smoke 连续 3 次通过
- 无阻塞级（P0）问题

---

## Phase 1：关键功能闭环（I2C + UDMA + APB）

### RTL完善
- I2C：补齐基础控制流程（start/stop/repeated-start 框架）
- UDMA：完善通道控制、完成中断、边界长度行为
- APB寄存器：复位值与字段行为统一

### UVM验证
- I2C 基本读写、repeated-start
- UDMA 基础搬运与中断检查
- APB R/W、reset value 检查
- 引入基础 scoreboard 与 predictor

### 交付物
- 用例：`tc_i2c_basic_rw`、`tc_i2c_repeat_start`、`tc_udma_basic`
- 缺陷清单与修复闭环记录

### Exit Criteria
- 关键功能用例全部通过
- 功能覆盖率达到阶段目标（>70%）

---

## Phase 2：系统闭环（中断 + 低功耗 + E2E）

### RTL完善
- PMU/RTC/WAKE 路径一致性
- INTC 的屏蔽/挂起/清除路径完善
- 顶层中断映射与唤醒原因寄存器联动

### UVM验证
- `RUN/SLEEP/DEEP-SLEEP` 切换与唤醒
- 多中断并发与优先级行为
- 端到端场景：I2C采集 -> UDMA搬运 -> 融合事件 -> 中断/上报

### 交付物
- 低功耗场景回归报告
- E2E 用例报告

### Exit Criteria
- 低功耗关键场景通过
- E2E 稳定通过（多seed）

---

## Phase 3：异常/随机/覆盖收敛

### RTL完善
- 补齐异常路径（NACK/timeout/busy冲突）
- 修复随机回归暴露的 corner case

### UVM验证
- 异常注入测试
- 大规模随机回归
- 功能覆盖 + 代码覆盖收敛

### 交付物
- 覆盖率报告（func/code）
- 未覆盖项说明与风险评估

### Exit Criteria
- 阶段覆盖目标满足（示例：func>90%, line>90%, fsm>95%）
- 无未关闭P0/P1

---

## Phase 4：实现签收与封板

### RTL完善
- 清理综合告警与不可综合结构
- 对齐综合约束与形式检查假设

### 验证/实现
- DC 综合 QoR
- FM 等价检查
- ECO 后最小回归

### 交付物
- 综合报告、FM报告、签收清单

### Exit Criteria
- FM 等价通过
- 阶段签收评审通过

---

## 5. 任务分工建议

- 设计主线：IP核心逻辑与顶层集成
- 验证主线：UVM环境、测试、覆盖率与回归
- 共同接口：寄存器规范、中断/低功耗时序

---

## 6. 缺陷优先级与处理SLA（建议）

- P0：24小时内定位并给出修复方案
- P1：48小时内闭环
- P2：按迭代节奏并入
- P3：版本尾部清理

---

## 7. 下一步执行（立刻）

1. 在 `sim/uvm/` 下创建 Phase 0 最小骨架
2. 先落地 `APB smoke` 测试链路
3. 建立第一版回归命令与报告模板

> 执行策略：每个阶段采用“RTL增量 + UVM增量 + 回归增量”并行推进，不等待完整版再验证。
